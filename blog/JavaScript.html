Blog Post #6

Hashes vs. Objects

December 28, 2015

For the last three weeks we've been learning about Ruby to perform simple tasks, but this week we've begun learning JavaScript. Not related at all to the language, Java, JavaScript was named simply as a marketing ploy to increase interest in the language at its creation. We've learned a little about how JavaScript works, and how it's very similar to what we've learned in Ruby. They have similar structures and use a lot of the same types of data such as strings, arrays, and integers, and they can all be managed in the same ways. One major difference in data storage in these two languages is that while Ruby uses hashes for key/value data pairing, JavaScript uses objects to store multiple layers of information.

We all know how hashes in Ruby work, they have keys that return values and thos values can be any type of class object. JavaScript's objects can also hold attributes with values. JavaScript has some more mobility in this area, as the object can also be treated similarly to Ruby's classes. JavaScript objects can contain variables that can be accessed and manipulated, as well as functions that use the object's stored data. This feature is similar to the way we can add attribute accessors and custom methods to our classes in Ruby.

Here's a short example of a JavaScript object that houses attributes and functions. It comes from one of this week's challenges, "Design a Basic Game".

In this code, I have created an object called "partner" to go along with the hero of the game that I designed. The partner has attribute values for a name, strength, health, and whether they are alive or not. There are also a few functions that can be done by the partner like searching the room for items, and subfunctions specifying effects for what the partner had found in the room. They can all be called with "partner.attribute" or "partner.function()"

In Ruby, you have to add an attribute modifier or accessor for your attributes to be accessible outside of your class, but in JavaScript, they're all accessible for reading and manipulation. They can even be modified and referred to by other objects by calling them with the syntax, "object.attribute", just as you do with accessible attributes in Ruby.

I think it's fair to say that JavaScript's objects are more versatile than Ruby's hashes in the sense that there are more types of data that can be stored in them. While objects can house variables, strings, arrays, and integers the same way hashes do, they can also house functions that can be applied to the data it stores and manipulate other objects.
