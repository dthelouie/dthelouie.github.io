Blog Post #2

HTML and CSS, Classes and IDs

November 27, 2015

Two more weeks with Dev Bootcamp, and we already have our own websites! We've learned a lot about HTML and CSS and how to use them together to begin to create websites like the ones we see every day. It's pretty awesome that we've gotten through so much material in such a short time. I know we've barely scratched the surface, but to think that I didn't know any of this a couple weeks ago really excites me. We learned about a few different tools and concepts in CSS this week such as the difference between margins, borders and padding, different types of positioning, display properties, and how to use classes and ids. I'm going to be talking about classes and ids.

First let's go over the difference between classes and ids. Classes and ids are both used in CSS to refer to an element in HTML rather than going through the list of what the element is nested in. They both go right into the opening tag of an element and can be directly manipulated by CSS properties. The main difference between classes and ids are the way they're assigned. A specific id can only be used on a single element, and a single element can only be assigned a single id. Classes, on the other hand can be assigned to a set of elements that will receive the same CSS properties. Elements can also have multiple classes. You can even have elements with a class and an id.

So that brings us to the question, "How do we know when to use each one?!" Classes are used for applying CSS properties to multiple elements, so if you think that there's any possibility that you'll be applying the same set of changes to more than one element, you should consider making a class. If you want a quick shortcut specific element and you'll only be applying this set of CSS properties to this element, you should use an id.
