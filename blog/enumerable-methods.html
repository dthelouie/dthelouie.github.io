Ruby: Enumerable Methods

December 10, 2015

This week we got to have some more practice with Ruby. We learned how to modify arrays and strings with methods, and store more complex information with hashes. We also learned about object classes and how each class has specific methods that can be applied to them. From there, we learned how to make our own classes and how to define methods that could be used for them. Two of the methods I learned about were each, and map.

The each and map methods are very similar. They're both iterators, and they apply a block of code to each item in the object that they're being applied to. Their main difference is what happens after the block is done running. With the each method, after the block runs, it will return the original object without storing the block's results, but is usually used for its side effects. If you want to run the each method and you want to use the index of each element of the object as a factor in your results, you could use the method, each_with_index. The map method will return an array filled with the results of the block of code that's being applied to the object. Map can also be used as collect, as they have the same effect.

Notice how when using each, if you want to see the results of the method, you have to use a printing method, and then have the original array returned. When using map, the results of the method are returned without having to apply another block of code. This is a clear illustration of the difference between the two methods. While they are both iterators, each is used for its side effects, and map is used for its results.
