Git and Version Control

November 13, 2015

Version Control is a very important part of individual work as well as collaborative work in the world of programming. It allows us to keep track of changes that occur in our work, and gives us the option to use it in case we run into issues with our current code. With version control, we can maintain a stable version of what we're working on, as well as experimental code without affecting our main code with the help of branches. We can make different branches to see how different techniques or processes will run together and pick the one that works best.

Git helps us track changes by logging changes and storing each version of our updated work. These versions are called "commits." In each of these commits, we can see what has been added, removed or modified. You can view the current version of the file as well as previous versions, each with commit messages with a short description of what has been changed since the previous version.

GitHub is a great place to store code due to its accessibility and ease. We can push our local repositories to GitHub straight from our terminals, and we can clone forked repositories to our computers. It makes collaborative work easier by providing a central location to store and combine code rather than working on a single file that has to be passed around through a flash drive. It also works as a place for you to allow others to look at your code for learning experiences and critique.
