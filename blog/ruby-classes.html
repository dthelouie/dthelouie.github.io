Ruby: Classes

December 17, 2015

With this week's material, we got to learn some new concepts in Ruby. We learned more about making classes, as well as the methods that we can write to be applied to them and how to read, and modify instance variables from outside of the class. Before I go into examples, I'm going to talk about classes and what they are.

Classes tell us what kind of object we are looking at, and what type of methods we can use on these objects. For instance, this whole time we've been using strings, arrays, integers, and hashes in our code. These are all objects that belong to individual classes. If you were to look each one up on the Ruby Docs, you would find a whole page dedicated to each class featuring a list of all the methods you could use on each class with short examples of how to use them.

Classes also use a different type of variable, and these variables are available for use by all methods across the class. These are called instance variables. They are easily recognizable, as they all start with the "@" symbol. This allows other methods within the class to access the data that is stored on them, as well as the ability to manipulate their data. Classes can also make use of attribute readers, writers, and accessors. These attribute methods allow us to read and write the value of the instance variables they're connected to directly, and outside of the class. Based on the type of attribute method that a variable is connected to, it can only be read or written, or both. Using attr_reader will allow us to read the variable but not write to it, while attr_writer will allow us to write but not read it. Using attr_accessor will allow you to both. If you don't know how fantasy basketball works, you have a set of NBA players, and those players accumulate stats for you through the course of a week, but only if they're starting on your active roster. These are the same stats that they accumulate in their real-world games, so you have to start players on the days that they have games. The name of the league, "Retirement Year" is a fantasy basketball league that I'm a part of during this NBA season. The list of players is mostly made up of players that I have on my team, and the rest are players that I wish I had on my team, but anyway let's dive into this.

Before I initialize my class, I declare a few attribute methods. I used attr_reader for the league name, since that's something that I won't want someone to change, attr_writer for my favorite team in case I want to change my favorite team, and attr_accessor for my name and my team's name, so that if I ever decided to give my team away, the owner's name and the team name could easily be changed. When initializing my class, I add the instance variables that I am connecting my attribute methods to, and I create hashes for the team's roster (with arrays to note what positions they play), the team's active roster, and the bench. After this, I create my instance methods. The active roster and the bench is what we'll be modifying so that we can accumulate stats for the season, so I created methods for starting a player, benching players, adding players, and dropping players from our team. I also have methods that show us our full roster, our active roster, and our bench. These methods can be called exactly the same way we call methods on other class objects.

Now to demonstrate how instance variables and attribute methods work, I will call the owner name and the team name, and then I will change the owner name. Since these are instance variables that have attr_readers and attr_accessors attached to them, they will comply, but watch what happens if I try to ask what my favorite team is.

Notice how when I try to call my favorite team, we get a NoMethodError. This is because you can't read the data in an instance variable if it's attached to an attr_writer. You can change what the variable's value is, but you can't see it or call it as you would with instance variables that are attached to attr_readers or attr_accessors.
