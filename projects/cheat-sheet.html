My Cheat Sheet

Ruby

  • Ruby is a scripting language
  • Uses elements such as Hashes, Arrays, Strings, and Integers to accomplish tasks
  • Integers are whole numbers, as opposed to floats which can have decimals. Integers can be transformed into floats with the method ".to_f"
  • Strings are any text housed between two of the same type of quotation marks
  • Arrays are collections of other elements housed between two square braces. Arrays can even hold other arrays!
  • Hashes are similar to arrays, but while arrays use numbers for indexing data, hashes use any other element that leads to another element. Elements in hashes come in key/value pairs. They are housed in curly braces.
  • Variables are also an element that is used a lot in Ruby. Variables are name values that can be assigned to other elements. If you ask for the variable's name, it will give you the value that it's been assigned to.
  • We use if statements to execute certain blocks of code when a certain conditon is true, and if it's false, we use an else statement to execute another block of code. If/else statements can account for many different conditions in the same situation. (if condition, execute block. elsif other condition, execute other block. else, execute other block. end
  • The keyword "while" is used for looping, and often iterating. (while this condition == true, execute block of code. end.)
  • Methods can be created to avoid repeating a certain block of code multiple times. (def method_name(argument) execute block. end)
  • The method "each" can also be used for iterating
  • Each type of object has its own set of methods that can be applied to them, which can be found in the Ruby Docs at ruby-doc.org.
  • You can create new objects by declaring a class. You need to define an initialization method, and methods that can be applied to the class. You can then treat it like any other object with its methods.

JavaScript

  • JavaScript is very similar to Ruby, in the sense that it uses all of the same elements and can accomplish many of the same tasks.
  • Variables are declared with "var" (var array = [])
  • In most cases, Ruby uses the keyword "end" to finish statements. In JavaScript, blocks of code that are to be executed in statements are housed in curly braces.
  • Looping can be done with "while" (while (condition){execute this block})as it is in Ruby, or with "for"(for (init var; until condition == true; increment variable){execute block}.
  • Functions are defined similarly to the way methods are in Ruby, except they are set up the same way as variables. (function = func_name(argument){execute block}
  • Objects can be created to store information similar to the way that Ruby's hashes do. You assign a key to value, and when you call the key, the value is returned. Functions can be added to objects just like methods in Ruby classes.

HTML

  • HTML: HyperText Markup Language
  • HTML is a language that is used to create website pages. It is interpreted by browsers and places links, text, and images within tags to present content.
  • Individual types of content are housed between tags (<tag>CONTENT</tag>)
  • Heading tags such as h1, h2, h3, etc. are used to note the hierarchy of headings on the page
  • The "p" tag is used for large sets of text, such as the main articles of a page.
  • The "a" tag is used for links (<a href="link">"alt-text"</a>)
  • The "img" tag is used for presenting images on a page. (<img src="link or relative path" alt="alt-text"/>)
  • Unordered lists can be opened with the "ul" tag, and ordered lists with the "ol" tag. Elements within the list are noted with the "li" tag.
  • New semantic tags in HTML5 are used for clearer notation of what the content is. These tags include "article", "aside", "footer", "header", "main", "nav", "section", and "summary".

CSS

  • CSS: Cascading Style Sheets
  • CSS is used to position and style elements in an HTML page.
  • CSS is held in a stylesheet, separately from the main HTML page.
  • Some styling properties that could be applied are color, font-family, font-size, height, width, background-color, text-decoration, border-radius, border, text-align, display, position, margin, opacity, font-weight, or float.
  • Properties that use size or space can be measured in pixels or em's
  • The syntax of CSS properties is selector{property: value;}
  • Classes and IDs can be applied to HTML elements to edit specific elements, or a group of elements.
  • Positioning properties include display, margin, border, padding, content, float, and clear

Git

  • Git is used for version control of code that we write. It keeps track of changes in documents that we write.
  • We use GitHub for a centralized location to push our code to, and efficiently promote collaborative work.
  • Git repositories can exist locally and remotely. We work on our individual computers and push to our remote repositories so others can access our files.
  • Changes are tracked by "commits" that are associated with each version of a document.
  • All git commands start with "git" and are executed in the command line.
  • Branches are used to separate changes from the master version of the document.
  • Keep track of branches with "git branch"
  • Create and move to a new branch with "git checkout -b BRANCH-NAME"
  • Switch branches with "git checkout BRANCH-NAME"
  • Create a repository with "git init"
  • Check the status of changed documents with "git status"
  • Commit changes with the commands, "git add .", "git commit -m 'COMMIT MESSAGE'"
  • Track commits with "git log"
  • After committing a change, push it to a remote repository with "git push origin REPO-NAME"
  • Merge branch in a repository to the master branch when ready on GitHub, and then pull to keep your local repository up to date.
  • Fetch and merge changes with "git pull"
  • Delete unwanted branches with "git branch -d BRANCH-NAME"

SQL

  • SQL: Structured Query Language
  • SQL is used for fetching information from databases.
  • Data is held on tables
  • Create a new table with "CREATE TABLE"
  • Add a new row with INSERT category-name INTO table-name
  • Gather information with SELECT category-name FROM table-name
  • Edit rows in tables with UPDATE.
  • Change existing tables with ALTER TABLE.
  • Delete rows in tables with DELETE FROM
  • The WHERE operator filters results of a query based on specified conditions
  • LIKE and BETWEEN are used in the WHERE clause.
  • ORDER BY (ASC and DESC) will sort results in ascending or descending order
  • LIMIT will specify the maximum number of rows that will be returned by a query.
